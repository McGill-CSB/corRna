CorRNA

Updated: Jan 23, 2011

INSTRUCTIONS

Step 1: Input an RNA sequence
Step 2: Choose a heuristic method
Step 3: Choose a search method (if applicable)
Step 4: Modify extra options (optional)
Step 5: Click Submit

DESCRIPTION

CorRNA - Predict and analyze deleterious mutations

CorRNA searches the mutational landscape for deleterious mutations. The program can use two different search algorithms (RNAMutants and FixedCGSampling) as a base method for exploring the mutational landscape. Two different search heuristics (structural and location) can also be used to refine the search. The program will then ouput the mutations along with a correlation value and an optional significance value.

CONSTRAINT METHODS

No Constraints
This option will run the algorithm without any structural or mutational heuristics and is useful to determine the effectiveness of the following constraints methods.
Structure Heuristic
This heuristic uses structural constraints to guide the exploration of the mutational landscape. CorRNA will first use the base pairing probability matrix generated by RNAFold to find base pairing locations that are not used in optimal structure. The program will then run the search algorithm while constraining the search to mutations that would have this structure as their optimal confirmation.
Mutation (location) Heuristic
This heuristic controls the locations at which mutations can occur in the sequence CorRNA preform successive runs of the search algorithm and progressively restricts locations along the sequence that have already been explored.

Search Algorithm

RNAmutants
This option runs the original RNAmutants algorithm
fixedCG Sampling
This option runs the novel algorithm which computes multiple mutations while preserving the G+C content of the wild sequence
Both
This option will run both RNAmutants and fixedCG Sampling

ADDITIONAL OPTIONS

Measure Significance of Deleterious Structure
This option runs a bootstrapping method where all mutations (single and multiple) are ranked according to their deleterious effect and their statistical significance is estimated.
RNAfold dangling end energies
This option allows you to control how RNAFold treats "dangling end" energies for bases adjacent to helices in free ends and multi-loops.
"With (-d1) only unpaired bases can participate in at most one dangling end, this is the default for mfe folding but unsupported for the partition function folding. With -d2 this check is ignored, dangling energies will be added for the bases adjacent to a helix on both sides in any case; this is the default for partition function folding (-p). -d or -d0 ignores dangling ends altogether (mostly for debugging).
With -d3 mfe folding will allow coaxial stacking of adjacent helices in multi-loops. At the moment the implementation will not allow coaxial stacking of the two interior pairs in a loop of degree 3 and works only for mfe folding." (RNAFold Manpage)

OUTPUT

A sortable table is displayed with the sequence mutation, mean free energy, optimal secondary structure, correlation coefficient of the structural ensemble, and statistical significance of the mutations. A web-link to the results is provided at the time of data submission which will report the status of the job. If the user provides an email address, this link and the results will also be sent by email.
