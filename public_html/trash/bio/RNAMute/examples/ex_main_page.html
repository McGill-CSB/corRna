Example

How to observe the results?

Examine the RNA sequence:   CGGCUCGUCGCAUCCUGGGGCUGAAGAAGGUCCCAAGGGUUGGGCUGUUCGCCCAUUAAAGCGGCACGCGAGCUGG
This sequence is taken from the 23S ribosomal RNA of the T. thermophila [1]. The sequence was cut between nucleotides 2507 - 2582.

First, the program should be executed in one of the two ways.
Instructions: how to run the program

Running the program with GUI (Graphical Users Interface):
After the program terminates to process the data (the "Start" button is released) the results can be viewed by pressing the "Result" button. When this button is pressed a file called "RESULT_TABLE.html" will open in a new window.
 

Running the program without GUI:
When the application terminates, which can take a few minutes, a new file called "RESULT_TABLE.html" will appear in the current directory (e.g.  ~/RNAmute). The results can be viewed by opening this file.  

 

 The HTML file that contains the results looks like this .
 To further explore the results follow the clickable blue text.


 In this page there are three tables:

·  The first table divides all new structures (that were accepted from all point mutations) to groups according to their second eigenvalue of the Laplacian matrix [2]. The second column presents how many vertices the structure in each group contains. In the third column - "Wild type Vertices", the group that holds the wild type is marked with "WT", and groups that have the same number of vertices as the wild type are marked with "*". The last column indicates how many structures with the specified "Second eigenvalue" exist.

Each value in the first column can be pressed in order to view the list of mutations corresponding to this value and the specified vertices number. For example, clicking on eigenvalue 0.381966 (with 5 vertices) will open the following table: click here

·  The first column lists the point mutations.

·  The second column contains the coarse-grain tree-edit distance of the mutation from the wild type.

·  The third column presents the Minimum Energy (in Kcals/mol) of the secondary structure of the mutants.

·  The fourth column contains the Shapiro representation [3,4] of the mutated sequences.

Additional information about each mutation can be obtained by clicking on the mutation's specifier. For example, for mutation C21G is: click here 

 

The page contains:

  1. Drawings of RNA secondary structures of WT sequence and mutated sequence.

  2. Option to download both plots in ps format (plots obtained by using the Vienna RNAfold program).

  3. The WT sequence and the mutated sequence.

  4. Second eigenvalues of the WT secondary structure and of the mutant secondary structure.

  5. The WT free energy and the mutant free energy (in Kcal/mol).

  6. The Shapiro representation of the secondary structure of both WT and mutant.

  7. Distance (according to the Shapiro representation) of mutant from WT and the average Shapiro distance of all mutants.

  8. Dot-bracket representation of secondary structure of both WT and mutant.

  9. Distance (according to dot-bracket representation) of mutant sequence from WT and the average dot-bracket distance of all mutants.

                      

To get back to the first page: "RESULT_TABLE.HTML"

·   The second table divides all new structures to groups according to their "Dot-bracket distance" from the wild type structure (the distance was calculated between dot-bracket representations of WT and mutations). The first column holds the distance's ranges that where calculated according to "Clustering resolution" for "Dot-bracket representation", which is set to 4 by default and can be changed by the user. Clustering resolution of X means that distances are sorted in each group and there is no two sequential distances in one of the groups that the difference between them is more or equal to X. Also, if there are two distances such that the difference between them is less than X, these distances are in the same group. The second column indicates how many structures in this "Distance range" exist. Clustering Resolution of groups in the second and third tables are set to 4 and 6 respectively, and can be changed in the GUI by the user.

Each distance range in the first column can be clicked on in order to view the list of mutations that has a distance value in that range. For example, mutations table obtained by clicking on the range 38.0-38.0 can be viewed here. This distance range includes only two mutations and is interesting because it contains structures of mutations with relatively large dot-bracket distances from the WT.

·  The first column lists the point mutations.

·  The second column presents the second eigenvalue of each mutation. Mutations are sorted by this column.

·  The third column presents the number of vertices of the tree graph of each mutation's secondary structure.

·  The fourth column presents the Minimum Energy in Kcals/mol of each mutation folded structure.

·  The fifth column shows the dot-bracket representation of the mutation secondary structure.


As before, each mutation can be clicked on to obtain more information. Additional information about each mutation can be found by clicking on the mutation name and the information page that will be obtained looks similar to what was shown before. In our case, these are the same two mutations as were obtained from the first table (eigenvalue 0.381966), and these are the only mutations in the run which break one of two hairpins and linearize the structure.

 

·   The third table is similar to the second table but it groups structures according to their Shapiro distance which is obtained from two Shapiro representations of the WT and mutation structure.  It is possible to see that the third table also groups some two mutations with relatively large distances to a separate category, and these two mutations are exactly the same mutations that were found in the "Eigenvalue table" and "Dot-bracket table".

 

 

    From the example we can conclude that the RNAmute package was able to find mutations that rearrange the secondary structure of the WT. It divides these mutations to separate categories in all tables: in the first table these mutations fall into a category with specific eigenvalue; in the second and third table these mutations fall into categories using graph edit distances.
