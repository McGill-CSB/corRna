Overview

RNAMute is an application that provides a user friendly interface for analyzing the effects of point mutations on RNAs secondary structure. As with proteins, mutations in RNAs have the potential to alter their functionality in important biological processes. When attempting to develop models for the prediction of conformational rearranging mutations in RNAs, useful structural information is available from folding algorithms that predict the secondary structure of RNAs based on energy minimization. Detecting mutations with desirable structural effects among all possible point mutations may then be valuable for the prediction of selective mutations that can be tested experimentally. The mutation prediction method is independent of the folding algorithms but relies on their success to predict the folding of small RNA structures. For convenience in this work, we chose to use the Vienna RNA package for the prediction and comparison of RNA secondary structures. The Vienna package already contains several utilities that are related to this work and can be used as building blocks in our mutation predictions.

The Package Content

1. mute_single - performs all possible "single point mutations" in an RNA sequence. It predicts the secondary structure of the wild type and mutants using Vienna's RNAfold, then calculates several different representations and similarity measures between the wild type and mutants, and finally produces a "result" file from the results obtained.

2. RNAmute.java - the main routine. Creates a "friendly" interface for the user. Receives as input a file with an RNA sequence, runs "mute_single", and generates an HTML file "RESULT_TABLE.html " that contains all the processed data from the "result" file organized in various tables.

3. calcEig2 - calculates the second smallest eigenvalue of the Laplacian matrix for each single point mutation.

4. b2Shapiro - converts the full structure from bracket notation to the weighted coarse grained notation introduced by Bruce Shapiro. This routine uses a function that is located in the Vienna package's "lib" directory.

5. runRnaMute - similar to RNAmute, but enables the user to insert the RNA sequence in a text area of the GUI instead of using a file.

 

  • Programs taken from the Vienna RNA package:

    1. RNAfold - predicts minimum energy secondary structures and base pairing probabilities.

    2. RNAdistance - calculates the distance between two RNA secondary structures represented as dot-bracket strings.

    While the program runs, a new directory  called "htmlDir" will be created. This directory contains all the HTML pages and all the drawings of the RNA secondary structures that are being calculated.

    Preparation and Compilation

        RNAMute can run only on a Linux platform, so all preparations and compilations that will be mentioned should be performed on a Linux platform with Java and "GNU CC" compiler installed. RNAMute has all its components already compiled and may be used without any compilations as described in the "How to Run the Program" part, but it has some components written in C that in some architectures may not work. In such a case, the Vienna RNA package should be downloaded from the web at http://www.tbi.univie.ac.at/~ivo/RNA/ and directory "ViennaRNA-1.4\lib" should be compiled by running the command "make" in this directory. All files from the directory "RNAMute\RNAMute_progs" should be copied to "ViennaRNA-1.4\Progs" and compiled with "makefile". "Makefile" that appears in the "ViennaRNA-1.4\Progs" directory should be overwritten. After the compilation finishes, files: "b2Shapiro", "calcEig2", "RNAdistance", "RNAfold" and "mute_single" should be copied from "ViennaRNA-1.4\Progs" directory to the "RNAMute\bin" directory. All files that are already in it should be overwritten. Make sure that all files in the "RNAMute\bin" directory are in an executable mode. If not, change their mode by typing the command:  >chmod 700 file_name ,when file_name is each file  from the list above. At this point, the RNAMute package can be used as described in the "How to Run the Program" part.

    Requirements

    This program can run on a Linux platform only. For using this program,  one needs to have "Java VM" and "GNU CC" compiler installed on the computer.

    For more information about downloading and installing  Java  click here.
    1. Please make sure that the directory that contains the program (e.g. ~/RNAMute) and all subdirectories have the correct Read/Write permissions for all users working with this program.
    2. RNAMute writes some necessary and temporary files while running. Thus, please make sure that you have enough  memory for placing those files. It is recommended that you will have approximately 10.0 free MB for processing sequences that are about 100 nt long.

    How to Run the Program

    Before running the program make sure that all the preparations are done.
    Open a Unix shell, and go to the directory that holds the program (e.g. ~/RNAMute).
    The RNAMute program can be executed in two ways:
    1. With Graphical User Interface (GUI) :

    • type the command:
      >java -jar runRnaMute.jar

       The GUI that appears after running the command looks like this:

        At this point, an RNA sequence may be inserted into the text box of the GUI.

    • The "Reset" button removes the sequence from the text area.
    • To start a calculation the "Start" button should be pressed.
    • During the calculation the "Start" button remains pressed and after the calculation stops, the "Start" button releases (please be patient as it may take a few seconds to generate the results).
    • Results may be observed by pressing the "Result" button.
      • If the file could not be opened by pressing the "Result" button, open the file "RESULT_TABLE.html", which is located in the same directory (e.g.~/RNAMute).

    2. Without Graphical User Interface (GUI) :

    • Type your sequence in a file (e.g., my_seq).
    • Save this file in the same directory where the program resides (e.g. ~/RNAMute)
    • From this directory, run the program "RNAmute", with the sequence file as an argument. Use this command:
      > java   -jar RNAmute.jar   my_seq
    • Wait until the application terminates (it may take a few seconds).
    • A new file "RESULT_TABLE.html" will appear in the current directory (e.g.  ~/RNAMute). To examine the output, open the file and follow its links.

    Examination of the Results

    The result file arranges the data of all point mutations in a convenient way. You can click the links in the table's cells to get  more information about each mutation.

    The easiest way to understand how to use the result file is to look at an example.

    Having some problems?

    First of all, make sure that all the preparations were performed and all the requirements have been fulfilled.

    If one of the single point mutations results in a completely linear structure (i.e., a single strand with no base pairing), the "Dot-bracket table" may not be generated and an exception error may appear. In such a case, inspect the "Eigenvalue table" and verify that indeed one of the mutations is causing a single strand.

    If you have some problems running the program with the Graphical Interface, try to run it without. For instructions go to "How to run the Program".
